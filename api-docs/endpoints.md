# Internal API Endpoints

This document details the internal REST API endpoints available in the TruthLens backend.

---

## 1. Health Check
* **Endpoint:** `/api/health`
* **Method:** `GET`
* **Purpose:** Instantly verify that the backend server is active and reachable by the frontend.
* **Response:**
  ```json
  {
    "ok": true,
    "message": "Backend is running"
  }
  ```

---

## 2. Create Analysis
* **Endpoint:** `/api/analyses`
* **Method:** `POST`
* **Purpose:** Submit text to the hallucination engine, analyze it, and save the result to MongoDB.
* **Request Body:**
  ```json
  {
    "originalText": "Python was created by Guido van Rossum in 1991. It is used by 100% of developers."
  }
  ```
* **Success Response (201 Created):**
  ```json
  {
    "success": true,
    "data": {
      "_id": "60d5ecb8b392...",
      "originalText": "Python was created by...",
      "score": 85,
      "confidenceLevel": "HIGH",
      "extractedClaims": [
        {
          "text": "Python was created by Guido van Rossum in 1991.",
          "type": "factual",
          "confidence": 0.6
        }
      ],
      "flaggedSentences": [
        {
          "text": "It is used by 100% of developers.",
          "reasons": ["Absolute Statement", "Unverifiable Statistic"]
        }
      ],
      "createdAt": "2024-03-27T10:00:00Z"
    }
  }
  ```
* **Common Errors:**
  * `400 Bad Request` → `originalText` is missing or empty.
  * `500 Server Error` → MongoDB insertion failure.
  * `503 Tunnel Unavailable` / `Network Error` → Generated by the frontend if the server is unreachable.

---

## 3. Get All History
* **Endpoint:** `/api/analyses`
* **Method:** `GET`
* **Purpose:** Retrieve all saved analysis records from the database, sorted by newest first.
* **Query Parameters (Optional):**
  * `search`: Filter results by text content.
  * `confidenceLevel`: Filter by `HIGH`, `MID`, or `LOW`.
* **Success Response (200 OK):**
  ```json
  {
    "success": true,
    "count": 15,
    "data": [
      {
        "_id": "60d5ecb8b392...",
        "originalText": "...",
        "score": 85
      }
    ]
  }
  ```

---

## 4. Get Analysis By ID
* **Endpoint:** `/api/analyses/:id`
* **Method:** `GET`
* **Purpose:** Retrieve the full details of a specific analysis report by its unique MongoDB ID.
* **Success Response (200 OK):** Returns a single analysis object.
* **Error Response:** `404 Not Found` if the ID does not exist.

---

## 5. Delete Analysis
* **Endpoint:** `/api/analyses/:id`
* **Method:** `DELETE`
* **Purpose:** Remove an analysis record from the user's history.
* **Success Response (200 OK):**
  ```json
  {
    "success": true,
    "message": "Analysis deleted successfully"
  }
  ```
